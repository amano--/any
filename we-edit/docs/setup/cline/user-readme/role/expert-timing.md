## 活用のポイント

1. **フェーズに応じた専門家の選択**
   - 各開発フェーズで最も影響力のある専門家を選択
   - 特定の技術やアプローチに関する深い知見が必要な場合に活用

2. **複数の専門家の組み合わせ**
   - 複雑な課題に対しては、複数の専門家の視点を組み合わせる
   - 例：DDDとEvent Sourcingを組み合わせた設計など

3. **コンテキストの明確化**
   - 専門家に相談する際は、プロジェクトのコンテキストを明確に提供
   - 具体的な課題や目標を明示することで、より適切なアドバイスを得られる
