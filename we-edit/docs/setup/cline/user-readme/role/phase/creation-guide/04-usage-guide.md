# エキスパートロール使用ガイド

## 1. 開発フェーズ別の活用

### 1.1 要件定義フェーズ
```markdown
1. 推奨エキスパート
   □ Karen McGrane（コンテンツ戦略）
      - 構造設計の視点
      - ユーザーニーズ分析
      - 再利用性の考慮
      - 拡張性の検討

   □ JoAnn Hackos（プロセス管理）
      - 文書化プロセス設計
      - 品質基準の設定
      - リソース計画
      - スケジュール管理

2. プロンプト例
   「Karen McGraneとして、以下の文書要件を分析してください：
    [要件の説明]
    
    以下の観点から評価と提案をお願いします：
    1. コンテンツ構造の適切性
    2. ユーザーニーズとの整合性

## 3. 効果的な使用方法

### 3.1 エキスパート選択
```markdown
1. 選択基準
   □ 文書の目的
   □ 対象読者
   □ 必要な専門性
   □ 期待する成果

2. 組み合わせ検討
   □ 主担当の決定
   □ 補助役割の特定
   □ 相互補完性
   □ 統合方法
```

### 3.2 プロンプト作成
```markdown
1. 基本要素
   □ エキスパートの明確な指定
   □ 専門性の強調
   □ 具体的な要件
   □ 期待する出力形式

2. 詳細化要素
   □ 背景情報の提供
   □ 制約条件の明示
   □ 品質基準の設定
   □ フィードバック方法
```

## 4. 成果の評価

### 4.1 評価基準
```markdown
1. 品質指標
   □ 専門的正確性
   □ 目的適合性
   □ 実用性
   □ 完全性

2. 効果指標
   □ ユーザー理解度
   □ 実装可能性
   □ 保守性
   □ 再利用性
```

### 4.2 改善サイクル
```markdown
1. フィードバック収集
   □ ユーザー評価

   □ 運用状況
   □ 問題点

2. 最適化
   □ エキスパート選択の見直し

   □ 要件の調整
   □ プロセスの改善
