## 4. 利用方法

### 4.1 基本的な流れ
1. 全体概要（00-overview.md）の理解
2. 文書構造（01-structure.md）の確認
3. 適切な専門家（02-expert-lists.md）の選択
4. テンプレート（03-templates.md）の活用
5. 使用ガイド（04-usage-guide.md）の参照
6. 事例（05-case-studies.md）からの学習

### 4.2 プロジェクトタイプ別
```markdown
1. 大規模プロジェクト
   - 包括的なアプローチ
   - 詳細な文書化
   - 厳格な品質管理
   - 組織的なレビュー

2. 中規模プロジェクト
   - バランスの取れたアプローチ
   - 必要十分な文書化
   - 効率的な品質管理
   - 効果的なレビュー

3. 小規模プロジェクト
   - 軽量なアプローチ
   - 最小限の文書化
   - 重点的な品質管理
   - 迅速なレビュー
```
