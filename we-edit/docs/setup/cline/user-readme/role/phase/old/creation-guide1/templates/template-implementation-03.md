# 実装・テストテンプレート

## 1. コーディングガイドライン

### 1.1 基本規約
```markdown
# コーディングガイドライン

## 命名規則
### 変数名
- キャメルケース使用
- 意味のある名前
- 一時変数は最小限
- 予約語回避

### メソッド名
- 動詞で開始
- 単一責務原則に従う
- 最大20行程度
- 引数は3つまで

### クラス名
- 名詞で開始
- 単一責任原則に従う
- インターフェースは形容詞
- 継承は慎重に
```

### 1.2 コード構造
```markdown
## コード構成
### ファイル構成
1. ヘッダー部
   - ファイル説明
   - 著作権表示
   - パッケージ宣言
   - インポート文

2. 実装部
   - クラス定義
   - フィールド定義
   - メソッド実装
   - 内部クラス

### コメント規約
1. ドキュメントコメント
   - クラス説明
   - メソッド説明
   - パラメータ説明
   - 戻り値説明
```
