# フェーズ別 資料作成手順書

## 1. 基本的な手順

### 1.1 資料作成の準備
1. フェーズと目的の明確化
2. 必要な資料の種類の特定
3. 適切なロールの選択
4. 具体的な文脈の整理

### 1.2 資料作成の実施
1. 適切なプロンプトの作成
2. 生成された内容の確認
3. 必要に応じた修正依頼
4. 最終的な取りまとめ

## 2. フェーズ別の特徴と推奨ロール

### 2.1 要求分析フェーズ
#### 作成される資料の例
- 要求仕様書
- ユースケース記述
- 要求トレーサビリティマトリクス
- ステークホルダー分析

#### 推奨ロール
1. Alistair Cockburn
   - ユースケース分析
   - 要求の段階的詳細化
2. Karl Wiegers
   - 要求工学の体系的アプローチ
   - 要求管理プロセス

#### 具体的な手順
```markdown
1. 要求の概要把握
   - Wiegerロールで要求の分類と優先順位付け
   - 具体的な文脈と制約条件の整理

2. 詳細分析
   - Cockburnロールでユースケース記述
   - ステークホルダー分析と関心事の整理

3. 文書化と検証
   - 要求仕様書の作成
   - トレーサビリティの確認
```

## 3. 効果的な資料作成のポイント

### 3.1 文脈の明確化
- プロジェクトの状況
- チームの構成と経験
- 技術スタック
- 制約条件

### 3.2 段階的な作成
1. 概要レベル
   - 全体像の把握
   - 主要な方針の決定

2. 詳細レベル
   - 具体的な実装方法
   - 具体例の提示

3. 補足情報
   - 参考資料
   - FAQ
   - トラブルシューティング

### 3.3 レビューと改善
- チームでのレビュー
- フィードバックの収集
- 継続的な更新

## 4. トラブルシューティング

### 4.1 一般的な課題と対処法
- 抽象的な内容への対処
- 実現可能性の確認
- チーム理解度の差の解消

### 4.2 フェーズ特有の課題
- 要求の曖昧さ
- 設計の複雑さ
- 実装の技術的制約
- テストの網羅性
- 運用の効率性

## 5. 成功のための推奨事項

1. **準備の徹底**
   - 目的の明確化
   - 必要な情報の収集
   - 適切なロールの選択

2. **段階的なアプローチ**
   - 概要から詳細へ
   - フィードバックの活用
   - 継続的な改善

3. **チームとの協働**
   - 早期のレビュー
   - 知見の共有
   - 改善の反映
